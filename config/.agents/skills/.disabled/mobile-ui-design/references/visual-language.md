# Visual Language

## タイポグラフィ

- 文字サイズではなく役割で階層を作る。タイトル、セクション、本文、補助、注記の役割を分ける。
- 1 画面で使うテキストスタイルは増やしすぎない。
- 強調は太さ、サイズ、余白、色の組み合わせで作り、全部を同時に強くしない。
- 長文を置く画面では、可読性を最優先する。ブランドの癖は見出し側へ寄せる。

## 色

- 色の仕事を決める。主要行動、状態表示、注意喚起、背景分離を混ぜない。
- ブランドカラーは主役にする場所を絞る。全要素に塗らない。
- 成功、警告、危険、無効、選択中の意味は一貫させる。
- コントラスト不足を「繊細さ」で正当化しない。

## 余白と密度

- 余白は高級感の演出ではなく、理解のために使う。
- 一覧や設定画面では密度を上げすぎず、情報探索を妨げない範囲で揃える。
- 近い要素は近く、意味が変わる境界ではしっかり離す。

## アイコンと画像

- アイコンだけで意味を伝えようとしない。主要操作にはテキストも検討する。
- 装飾画像は文脈を補強するときだけ使う。情報や操作を押しのけない。
- ブランドイラストやグラフィックは、初回体験や空状態の情緒づけに使う。

## 文言

- ボタンやラベルは、利用者が次に起こることを予測できる文にする。
- プレースホルダーは説明文の代わりにしない。
- エラーは原因と回復行動を短く示す。
- ブランドトーンは残してよいが、意味の明確さを削らない。

## ブランド世界観の落とし込み

- ブランドは配色だけでなく、緊張感、温度感、速度感、言葉遣いで作る。
- 高級感を出したいなら、静かな余白、抑制された色数、慎重なモーションを使う。
- 親しみやすさを出したいなら、硬すぎる文言や過密レイアウトを避ける。
- 実務系プロダクトでは、世界観より信頼感と読みやすさを優先する。

## モーション

- 状態変化、注目誘導、因果関係の説明にだけ使う。
- 速さと減速の性格でブランドを出してよいが、待ち時間を増やさない。
- すべての要素を動かさない。意味のある箇所だけ動かす。

## チェックポイント

- 主要アクションは 3 秒以内に見つかるか
- タイトル、本文、補助情報の階層が一目でわかるか
- 色の意味が画面ごとに変わっていないか
- ブランドの癖が可読性や操作性を傷つけていないか
