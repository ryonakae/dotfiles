# Claude Code セカンドオピニオン

## 概要

Claude Code CLI の非対話モード（`-p`）で直接実行して、外部視点でのセカンドオピニオンを得る。Claude Code はコードベースに直接アクセスでき、自律的にファイルの読み取りや検索を行える。

## 使い方

Bash ツールで `claude -p` を実行する。必要に応じて事前に Read / Grep / Glob でコンテキストを収集し、プロンプトに含める。

### コマンド例

```bash
# 基本形
claude -p "質問内容"

# パイプでコンテキストを渡す
cat src/file.ts | claude -p "このコードをレビューして"

# モデルを指定
claude -p "質問内容" --model sonnet

# 読み取り専用ツールに制限（セカンドオピニオン用途に推奨）
claude -p "質問内容" --allowedTools "Read,Grep,Glob"

# システムプロンプトを指定
claude -p "質問内容" --system-prompt "あなたはコードレビューの専門家です"
```

### 有用なオプション

| オプション | 説明 |
| --- | --- |
| `-p` / `--print` | 非対話モード（必須） |
| `--model <model>` | モデル指定（例: `haiku`, `sonnet`, `opus`） |
| `--output-format <format>` | 出力形式（`text`, `json`, `stream-json`） |
| `--system-prompt <prompt>` | システムプロンプトの指定 |
| `--allowedTools <tools>` | 使用可能なツールの制限（例: `"Read,Grep,Glob"`） |
| `--add-dir <dirs>` | 追加ディレクトリへのアクセスを許可 |

## いつ使うか

- 複数の実装方針で迷う時
- 重要な設計判断・アーキテクチャ選択の前
- コードレビューのダブルチェック
- ユーザーが「セカンドオピニオンが欲しい」とリクエストした時

## いつ使わないか

- 単純で明確な実装タスク
- 既に方針が決まっており追加意見が不要な場合

## 注意事項

- Claude Code は独立したプロセスとして実行される。現在の会話の文脈は引き継がれないため、必要な情報はすべてプロンプトに含めること
- 機密情報（API キー、パスワード等）はプロンプトに含めないこと
- Claude Code の回答は参考情報として扱い、最終判断はプロジェクトのコンテキストを踏まえて行うこと
